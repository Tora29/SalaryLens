現在の変更からPRを作成してください:

【手順】
1. 変更内容の確認と分析
   - git status で変更ファイルを確認
   - git diff --stat で変更量の概要を把握
   - 変更内容を分析してPRの目的と影響範囲を理解

2. ブランチ名の自動生成
   変更内容から適切なブランチ名を生成（ブランチ命名規則に従う）:
   - feature/ - 新機能追加
   - fix/ - バグ修正
   - refactor/ - リファクタリング
   - docs/ - ドキュメント更新
   - test/ - テスト追加・修正
   - chore/ - ビルドプロセスやツール変更

   例: docs/enhance-architecture-remove-inventory

3. ブランチ作成
   mainブランチから新しいブランチを作成
   `git checkout -b <ブランチ名>`

4. コミット作成
   変更をステージング・コミット
   - コミットメッセージ形式:
     ```
     <type>: <簡潔な要約（日本語）>

     - <変更内容1>
     - <変更内容2>
     - <変更内容3>

     🤖 Generated with [Claude Code](https://claude.com/claude-code)

     Co-Authored-By: Claude <noreply@anthropic.com>
     ```
   - typeはブランチのプレフィックスと一致させる

5. リモートにプッシュ
   `git push -u origin <ブランチ名>`

6. PR作成
   gh pr create でPRを作成（以下の形式で本文を生成）
